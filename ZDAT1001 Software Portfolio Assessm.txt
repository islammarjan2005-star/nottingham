Task Description
You are required to complete the following tasks using the provided dataset:

1. Database Design & Implementation
Design a suitable relational database structure.
Define appropriate tables, columns, and data types.
Implement data ingestion using Python and store the dataset in the database.
Use SQL queries to manipulate the data (avoid working with the Excel file beyond initial ingestion).
2. Data Analysis & Visualisation
Identify at least two key questions to explore using data analysis.
Implement reproducible and modular Python code (functions, classes, and scripts).
Produce clear, well-labelled visualisations (at least one per question).
Follow best practices for software development, including code structure, modularity, and documentation.
Example Questions for Analysis
(Refine these based on your Exploratory Data Analysis - EDA)

Do EC claims increase as deadlines approach?
Are there specific modules that generate more EC claims than others?
How does the EC claim rate change over time?
How does the response time for a claim vary over time?
Note: To achieve a C-band grade or higher, you must introduce at least one additional question beyond the examples provided.

3. Documentation & Reporting
Maintain clear software documentation (README.md, docstrings, comments).
Use Markdown (REPORT.md) for a concise 500-word report summarising findings.
Incorporate visualisations and annotations to enhance clarity.
4. GitHub Repository
Use the [provided GitHub Classroom repository}(https://classroom.github.com/a/Kwd6MkV0) to manage your project.
Ensure all files, code, reports, and documentation are committed to GitHub.
Follow best practices for Git usage and version control.
Project Structure
When you register using the GitHub Classroom link, a template project will be created. You are free to organise the repository as you see fit, but the following best practices are recommended:

Include two files at the top level: README.md and REPORT.md.
README.md should outline your project structure and provide necessary documentation, e.g., instructions to run code.
REPORT.md should contain the final report, including questions, write-up, and figures/tables.
Use Markdown features like links to reference folder locations or other sources. You may use Jupyter Notebooks and Python files, ensuring reproducibility and clear documentation.

Excluding Datasets from GitHub
Exclude large datasets or sensitive data from GitHub repositories.
Store datasets externally and reference them in your code.
Use a .gitignore file in the root of your repository and include:
/data
This prevents Git from tracking large files such as CSVs.
